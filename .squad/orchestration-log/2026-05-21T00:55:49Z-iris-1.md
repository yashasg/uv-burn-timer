## Deliverable

**File:** `.squad/decisions/inbox/iris-uvi-forecast-pushback-response.md` (merged into decisions.md)

**Key Decisions:**

### 1. Nomenclature — Kill tier labels
- **Prior:** Wheeler required Forecast / Outlook / Trend labels per row
- **Revised:** Single "Forecast" section header (Apple Weather consistent); `TierBadge` reserved for WHO severity only
- **Result:** Zero terminology collision; users recognize "Forecast" from iOS Weather

### 2. Visual treatment — Flat design
- **Prior:** Wheeler required opacity/gray demotion on days 6–10
- **Revised:** All 10 rows rendered with identical visual weight (headlines, body text, icon opacity)
- **Accessibility rationale:** Opacity-based decay at normal text size fails WCAG 1.4.3 unless opacity ≥80% — but at 80% the visual signal is negligible anyway
- **Precedent:** Apple Weather card uses flat treatment; HIG is silent; empirically strongest signal

### 3. Confidence disclosure — Footnote approach
- **Pattern:** Single `.caption .secondary` line at card bottom
- **Text:** "UV accuracy beyond 3–5 days decreases as cloud cover becomes harder to predict."
- **Satisfies:** Wheeler's §5.1 ("must be visible on the card, not just in About")
- **Accessibility:** `.caption` at full `.secondary` opacity passes WCAG 1.4.3 on `.regularMaterial` background
- **Precedent:** Apple Weather uses footnote-style attribution (weather.com lock-up)

### 4. Time-to-burn picker
- **Entry:** "Plan for another time" chip below main burn-time number
- **Sheet:** `DatePicker(displayedComponents: [.date, .hourAndMinute])` with `.graphical` style
- **Range:** `Date.now` to `Date.now + 10 days` (delta noted: Wheeler pushes back for D+7 cap)
- **Result display:** Rendered at `.title2` scale, same visual language as hero card
- **Edge cases:** UVI 0 → "no UV at this hour"; no skin type → existing D-2026-05-19-012 gate; forecast unavailable → retry prompt
